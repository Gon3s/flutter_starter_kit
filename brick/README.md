# flutter_starter_kit

A Flutter starter kit with Riverpod, GoRouter, Freezed, and optional Firebase + authentication.

## Usage

```bash
mason make flutter_starter_kit
```

## Variables

| Variable | Type | Default | Description |
|----------|------|---------|-------------|
| `app_name` | string | `my_app` | App name (snake_case) |
| `org_name` | string | `com.example` | Organisation identifier |
| `app_description` | string | `A new Flutter app.` | Short description |
| `avec_firebase` | boolean | `true` | Include Firebase (Analytics, Crashlytics, Messaging) |
| `avec_auth` | boolean | `true` | Include authentication (Email/Password via Firebase Auth) |

## What's included

- **Riverpod** — state management with code generation
- **GoRouter** — declarative routing with code generation
- **Freezed** — immutable data classes
- **AppStrings** — FR/EN localisation without external packages
- **Bootstrap** — error handlers, system UI, environment management
- **Common widgets** — `PrimaryButton`, `EmptyPlaceholder`
- **Utils** — `ColoredDebugPrinter`, `StringValidator`, `InMemoryStore`

### With Firebase (`avec_firebase: true`)

- Firebase Analytics, Crashlytics, Messaging
- Push notifications setup (foreground/background/terminated)
- `firebase_options.dart` pattern (gitignored, generated by `flutterfire configure`)

### With Auth (`avec_auth: true`, requires Firebase)

- Firebase Auth Email/Password
- Secure session storage (`flutter_secure_storage`)
- Sign In / Sign Up screens
- Account screen (user info, change password, delete account)
- Automatic routing based on auth state

## Post-generation steps

### With Firebase

1. Run `flutterfire configure` to generate `lib/firebase_options.dart`
2. Add secrets to GitHub Actions:
   - `FIREBASE_OPTIONS` — content of `firebase_options.dart`
   - `GOOGLE_SERVICE_INFO_PLIST` — content of `GoogleService-Info.plist`
   - `GOOGLE_SERVICES_JSON` — content of `google-services.json`

### Without Firebase

Just run the app:

```bash
flutter run
```
